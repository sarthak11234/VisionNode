### techStack.txt: Audition Agent (LAA) Implementation

**1. Core Intelligence (Inference Layer)**

* **Platform:** **Ollama** (v0.5.x or higher)
* **Vision Model:** `llama3.2-vision:11b` (Primary) or `qwen2.5-vl:7b` (Speed-optimized).
* **Purpose:** Local image-to-text conversion (OCR) and structured JSON extraction from handwritten/printed tables.

**2. Backend (Logic & Orchestration)**

* **Framework:** **FastAPI** (Python 3.12+)
* **Validation:** **Pydantic v2** (Strict typing for participant data).
* **Async Handling:** `httpx` (To communicate with Ollama's local API without blocking).
* **Persistence:** **SQLite** (via SQLAlchemy or Tortoise ORM) to prevent duplicate messaging and store gathering history.

**3. Frontend (User Interface)**

* **Library:** **React 19** (Vite-powered for instant HMR).
* **Styling:** **Tailwind CSS 4.0** (Utilizing the new zero-runtime engine).
* **Components:** **shadcn/ui** (Radix-based primitives) or **HeroUI** (for that high-gloss Dribbble aesthetic).
* **State Management:** **Zustand** (Lightweight alternative to Redux for handling the audition list state).

**4. Action & Automation (Messaging Layer)**

* **Automation:** **pywhatkit** (Version 5.4+) or **Selenium/WebDriver** (For more granular control over WhatsApp Web).
* **Rate Limiting:** Python `random` & `time` (To simulate human typing behavior and intervals).
* **Browser:** Google Chrome (Headless mode supported for automated background sending).

**5. Infrastructure & Dev Tooling**

* **Containerization:** **Docker + Docker Compose** (Bundling FastAPI and Ollama for one-click deployment).
* **API Documentation:** **Swagger UI** (Built into FastAPI at `/docs`).
